# Руководство для администраторов: Обработка Proposals

## Общий обзор

Данное руководство описывает процесс проверки и обработки предложений (proposals) в системе FaceBase. Администраторы должны следовать указанным шагам для обеспечения качественной модерации контента.

## Статусы Proposals

### Доступные статусы:

- `waiting_channel_approval` - Канал ожидает проверки (начальный статус любой заявки)
- `channel_approved` - Канал одобрен
- `channel_rejected` - Канал отклонен
- `waiting_attachments_approval` - Файлы интеграции ожидают проверки (второй этап заявки)
- `attachments_approved` - Файлы интеграции одобрены
- `attachments_rejected` - Файлы интеграции отклонены
- `waiting_approval` - Рекламная интеграция опубликована и ожидает проверки
- `approved` - Опубликованная рекламная интеграция одобрена, задание выполнено
- `rejected` - Опубликованная рекламная интеграция отклонена, задание не выполнено

## Концепции процесса обработки

### Получение списка новых proposals

1. Перейти в админ-панель
2. Открыть раздел "Proposals"
3. Отфильтровать по статусу `waiting_channel_approval`
4. Выбрать proposal для проверки

### Флоу проверки заявки

#### Шаг 1: Проверка канала

- Проверяем канал на соответствие требованиям задания

**Действия:**

- Проанализировать прикрепленный к заявке канал в поле `Channel Id`
- Если всё в порядке → перевести в статус `channel_approved` и указать срок реализации рекламной интеграции для следующего шага в поле `deadline`
- Если есть проблемы → перевести в статус `channel_rejected` и приложить комментарий(-ии) в поле `Status Details`

#### Шаг 2: Проверка содержимого интеграции

- Проверяем файлы и(-или) текст интеграции на соответствие заданным требованиям

**Действия:**

- Проанализировать содержимое файлов и текста интеграции в полях `Attachments Text` и `Attachments Files`
- Если всё в порядке → перевести в статус `attachments_approved`, указать срок публикации рекламной интеграции в поле `deadline` и приолжить **ERID** в поле `erid`
- Если есть проблемы → перевести в статус `attachments_rejected` и приложить комментарий(-ии) в поле `Status Details`

#### Шаг 3: Проверка публикации

- Проверяем опубликованную рекламную интеграцию

**Действия:**

- Проанализировать публикацию рекламной интеграции в соответствующей соц.сети
- Если всё в порядке → перевести в статус `approved`, произвести перевод средств пользователю
- Если есть проблемы → перевести в статус `rejected` и приложить комментарий(-ии) в поле `Status Details`

### Важно понимать

- Отслеживать стоит только заявки со статусом, у которого есть префикс `waiting_`. Это означает, что пользователь выполнил необходимые условия на текущем шаге и отправил какие либо данные на проверку.
- Если причин отказа несколько, перечислять необходимо путем добавления отдельных друг от друга айтемов, оно все автоматически соберется в список юзеру
